# Runtime Working Contract

1. Merged code, migrations, tests, and CI are implementation truth.
2. Product decisions come from the separately governed private plan database through a scoped vertical.
3. Never copy the planning database, private sources, prompts, or archive paths here.
4. Work in one small issue and PR at a time; keep inactive verticals unreachable.
5. Every mutation requires tenant scope, authorization, policy, evidence, and explicit billing treatment.
6. Preserve honest proof levels. Green unit tests are not E2E, release, or production proof.
7. Stop for unresolved legal, security, privacy, provider, destructive-data, or money decisions.

Validation command: `pnpm run ci`.
